# data/ — llmopt collaboration artifacts (untracked except this README)

Everything here is either supplied by llmopt or generated by axiom's
audit/gate tooling. None of it is source; all of it is reproducible.

## data/llmopt/ — supplied by llmopt (inputs of record)

| file | what | provenance |
|------|------|------------|
| `axiom_parity_farm.jsonl` | 74,426 farm rows (diff/equiv/equiv_mod_const tasks) for the oracle parity audit | regenerated dump, coefficient-fixed (parity run 2) |
| `axiom_parity_ref.jsonl` | sympy reference verdicts for the same rows | llmopt |
| `axiom_qual_roots.jsonl` | 480 Phase C qualification roots, 60/level L1–L8, fresh 69M seed band (contamination-clean) | llmopt |
| `markov_prior.tsv` | rule-bigram prior (48 unigram `U\tname\tcount`, 182 bigram `B\tprev\tnext\tcount`) exported from checkpoints/markov_prior.json | llmopt |

## data/parity/ — oracle audit outputs

Produced by `build-rel/axiom-oracle.exe data/llmopt/axiom_parity_farm.jsonl
data/parity/axiom_verdicts.jsonl` and the scoring script. Record of note:
runs 1+2 in `docs/specs/2026-07-18-parity-run-1.md` (PASS, 0 crossings,
tax 3.67%). `parity_ref_findings.jsonl` = the 64 adjudicated ref-side
errors (run 1); `parity_tax_rows.jsonl` = UNDECIDED inventory;
`stage2_*.jsonl` = diff-text-mismatch equivalence pass.

## data/qual/ — Phase C qualification gate runs

Produced by `build-rel/axiom-qual-gate.exe data/llmopt/axiom_qual_roots.jsonl
<max_level> <budget>`. TSV columns: id, level, verdict
(solved/unsolved/DIFFBACK-FAIL/REPLAY-FAIL), nodes, seconds, answer sstr.
One file per rules tranche so before/after compares stay trivial.
`tranche1_l1_4`: L1 60/60, L2 60/60, L3 49/60, L4 14/60, 0 soundness
failures, budget 200 width 8. Official parity reference = llmopt's sympy
run on the same roots (pending).
