# axiom Phase B Implementation Plan (mathgen port: generator parity with llmopt)

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:executing-plans. TDD: RED before implementation on every task.

**Goal:** Port llmopt's problem generator (`llmopt/mathgen/problems.py`, levels L1–L8) to axiom, gated by *exact-expression parity*: for 10⁴ (level, seed) pairs, axiom's generated expression must equal the sympy generator's sstr byte-for-byte (or the level is listed intentionally-diverged with a fresh contamination audit). Per llmopt's `docs/superpowers/specs/2026-07-18-axiom-backend.md`; llmopt owns adjudication.

**Architecture:** Three layers, each with its own parity gate, hardest dependency first:
1. `ax::pyrand` — bit-exact CPython `random.Random`: MT19937 core, **string seeding** (mandatory — llmopt's tuple-hash reproducibility scar) via UTF-8 bytes + SHA-512 digest → little-endian 32-bit words → `init_by_array`; `random()` (genrand_res53), `getrandbits`/`_randbelow` rejection sampling, `randint`, `choice`, `shuffle` (reversed Fisher–Yates). SHA-512 implemented in-house (STL has no crypto; FIPS 180-4, ~100 lines, test vectors from the standard).
2. `ax::sym` sstr-order printing — sympy's default Add/Mul print order (its canonical sort key), needed for byte-exact output. Scoped to the mathgen expression zoo, not sympy-general.
3. `ax::mathgen` — `_atom` / `_expression` / `_expression_l4..l8` ported line-for-line, including the `f"l6-sub-{rng.random()}"` sub-seeding (requires Python-repr float formatting: shortest round-trip, `std::format("{}", d)` — verified in the fixture gate, not assumed).

**Oracle discipline:** No hardcoded expected values. Fixtures are generated by `scripts/gen_pyrand_fixture.py` / `scripts/gen_mathgen_fixture.py` (run against local CPython 3.14 + sympy on demand), checked in with provenance headers; C++ tests consume the JSONL fixtures. Regenerating fixtures on another Python version must be a no-op diff (both scripts assert the documented algorithm versions). **Pinned 2026-07-18: CPython 3.12.13 (llmopt WSL farm) and 3.14.4 (axiom dev) produce bit-exact pyrand fixtures.**

**File map:**
- `include/ax/pyrand/pyrand.hpp`, `src/pyrand/{sha512,pyrand}.cpp`, tests `tests/pyrand/`
- `src/sym/print.cpp` extension or `print_sstr.cpp` — sympy-ordered sstr printer
- `include/ax/mathgen/problems.hpp`, `src/mathgen/problems.cpp`, tests `tests/mathgen/`
- `scripts/gen_pyrand_fixture.py`, `scripts/gen_mathgen_fixture.py`

**Tasks:**
1. SHA-512 (FIPS 180-4 vectors: empty, "abc", 896-bit, 1M-a) + MT19937 + string seed + random/getrandbits/randint/choice/shuffle, fixture-gated (1k draws × 8 seed strings incl. the llmopt seed formats). Commit `feat(pyrand): bit-exact CPython random.Random - MT19937, string seeding, sha512`.
2. sstr-ordered printer, fixture-gated on mathgen-shaped expressions. Commit `feat(sym): sympy-ordered sstr printer`.
3. L1–L3 builders + parity fixture 10³ pairs. Commit `feat(mathgen): L1-L3 generator, exact parity`.
4. L4–L8 builders + the full 10⁴ (level, seed) parity gate; divergences documented per llmopt spec. Commit `feat(mathgen): L4-L8 generator, 10^4-pair parity gate`.

**Non-goals:** limit/complex/matrix/ODE problem kinds (not in the L1–L8 integrate/differentiate ladder llmopt asked for) until Phase C needs them; sympy-general printing.
