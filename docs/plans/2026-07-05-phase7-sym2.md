# axiom Phase 7 Implementation Plan (ax::sym part 2 — integration, solvers)

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:executing-plans. TDD: RED before implementation on every task.

**Goal:** Symbolic integration ("Risch-lite": table + linearity + u-substitution + integration by parts + partial fractions) and equation solving (polynomial exact through quartic where radical forms stay sane, Durand-Kerner numeric fallback, unary-function isolation, symbolic linear systems).

**Architecture:** Integration returns `std::optional<expr>` — `nullopt` means "couldn't integrate", never a wrong answer. Verification oracle throughout: `diff(integrate(f)) - f` evaluates to ~0 at random points (no invented antiderivative strings). Solvers return exact `expr` roots when obtainable (rational-root deflation, quadratic formula, Cardano with the trig form for casus irreducibilis, Ferrari when the resolvent cubic has a rational root), else `std::complex<double>` roots via Durand-Kerner; every exact root is verified by numeric substitution. Linear systems: Gaussian elimination over expr with exact rational pivots when entries are rational, expr-division pivots otherwise.

**Tech Stack:** C++23 STL, GoogleTest, existing ax::sym (expr, poly, diff), ax::core rational.

**File map:**
- `src/sym/expr.cpp`, `src/sym/calc.cpp`, `src/sym/print.cpp` — extend fn set: atan, asin, acos (eval, derivative, LaTeX)
- `include/ax/sym/expand.hpp` + `src/sym/expand.cpp` — polynomial expansion
- `include/ax/sym/solve.hpp` + `src/sym/solve.cpp` — poly roots (exact + numeric), equation solving, linear systems
- `include/ax/sym/integrate.hpp` + `src/sym/integrate.cpp` — antiderivatives
- Tests: `tests/sym/expand_test.cpp`, `solve_test.cpp`, `integrate_test.cpp`

### Task 1: fn extensions + expand

Extend the fn table (all three consumers):
- eval: `atan, asin, acos` → std::atan etc.
- diff: `atan → 1/(1+u²)`, `asin → 1/sqrt(1-u²)`, `acos → -1/sqrt(1-u²)` (chain rule as existing).
- LaTeX printer: `\arctan` etc. (text printer already generic name(arg)).

New `expand(e)`: distribute products over sums and expand integer powers of sums (multinomial via repeated distribution). Canonicalization then collects like terms. Non-polynomial subtrees (fn nodes, symbolic exponents) pass through untouched with expanded arguments.

```cpp
// include/ax/sym/expand.hpp
expr expand(const expr& e);
```

Tests: diff(atan(x)) structurally 1/(1+x²) after simplify; eval(asin(0.5)) ≈ pi/6; numeric diff cross-check for all three (central difference, tol 1e-6); expand((x+1)²) same-node as x²+2x+1; expand((x+y)(x−y)) = x²−y²; expand((x+1)³) has coeff 3 on x²; expand(sin((x+1)x)) expands inside fn arg; expand leaves x²+1 fixed.
Commit `feat(sym): atan/asin/acos support and polynomial expansion`.

### Task 2: polynomial root solving

API (`namespace ax::sym`, header `solve.hpp`):

```cpp
/** Roots of a univariate polynomial equation p == 0. */
struct poly_roots {
  std::vector<expr> exact;                  ///< closed-form roots (verified)
  std::vector<std::complex<double>> approx; ///< numeric roots when exact fails
  bool complete_exact;                      ///< all roots are in `exact`
};

/** Solve p(x) == 0 for rational-coefficient p (from_expr must succeed).
    Exact ladder: rational-root deflation → deg<=2 quadratic formula (radical
    exprs, complex pairs omitted from exact but counted) → deg 3 Cardano
    (real radical form when D >= 0, 2√(-p/3)cos((acos(...)−2πk)/3) form when
    three real roots) → deg 4 Ferrari when the resolvent cubic has a rational
    root. Anything unresolved lands in approx via Durand-Kerner.
    Note: complex exact roots are not represented (expr is real-valued);
    complete_exact is false when nonreal roots exist beyond deg 2 deflation. */
poly_roots solve_poly(const expr& equation_lhs, const expr& x);

/** Durand-Kerner: all complex roots of arbitrary-degree p (double coeffs).
    Deterministic start points (0.4+0.9i)^k; iterates to |correction| < tol
    or max_iter. */
std::vector<std::complex<double>> durand_kerner(std::span<const double> coeffs,
                                                double tol = 1e-12,
                                                int max_iter = 200);
```

Quadratic with negative discriminant: real exprs impossible → the conjugate pair goes to approx (computed in closed form, not iterated), complete_exact = false.

Tests (oracle: substitute each exact root numerically, |p(root)| < 1e-9; approx roots likewise):
- linear 2x+3: exact {−3/2}, complete.
- quadratic x²−5x+6: exact {2, 3}; x²−2: exact {±√2} eval ≈ ±1.41421356; x²+1: exact empty, approx = ±i, complete_exact false.
- cubic with rational root x³−6x²+11x−6: exact {1,2,3}.
- cubic one real root x³+x−2 → rational root 1 deflates; x³+2x+1 (no rational root, D<0 one real): exact real root via Cardano radical form, eval satisfies |p| < 1e-9; conjugate pair in approx.
- casus irreducibilis x³−3x+1 (three real irrational roots): three exact trig-form roots, each |p(root)| < 1e-9.
- quartic x⁴−5x²+4: exact {±1, ±2} (biquadratic → resolvent path or deflation).
- degree 6 with no closed form x⁶−x−1: approx has 6 roots, each |p(z)| < 1e-9, complete_exact false.
- durand_kerner on (x−1)(x−2)(x−3) coeffs: converges to {1,2,3} any order, tol 1e-9.
- from_expr failure (sin(x)−1) throws std::invalid_argument.

Commit `feat(sym): polynomial solver — exact through quartic, Durand-Kerner fallback`.

### Task 3: integration — table, linearity, power rule

API (`namespace ax::sym`, header `integrate.hpp`):

```cpp
/** Antiderivative of e with respect to x (no +C). nullopt = cannot integrate.
    Guarantee: if a value is returned, diff(result, x) simplifies/evaluates
    back to e (verified property, the test oracle). */
std::optional<expr> integrate(const expr& e, const expr& x);
```

Core dispatcher (recursive):
- constants (in x): c → c·x.
- linearity: ∫Σ = Σ∫ (any term failing → nullopt); constant factors pulled out of mul.
- power rule: x^n (n rational ≠ −1) → x^(n+1)/(n+1); x^−1 → log(x).
- table on fn(u) with **u linear in x** (u = ax+b detected, result divided by a): sin→−cos, cos→sin, tan→−log(cos), exp→exp, sqrt(u)→(2/3)u^(3/2), log(u)→u·log(u)−u, 1/(1+x²)-shaped handled in Task 4 partial fractions, asin/acos/atan of linear args via parts in Task 4.
- pow of linear base: (ax+b)^n like power rule /a.

Tests (all pass the diff-roundtrip oracle at 8 random points, tol 1e-7, plus structural spot checks):
- ∫5 = 5x; ∫x³ = x⁴/4; ∫(3x²+2x+1) = x³+x²+x; ∫1/x = log x.
- ∫sin(2x+1) = −cos(2x+1)/2; ∫e^(3x) = e^(3x)/3; ∫(2x+5)⁷ = (2x+5)⁸/16.
- ∫x^(1/2) = (2/3)x^(3/2).
- ∫sin(x²) → nullopt (honest failure); ∫ w.r.t. x of y = y·x (other symbols are constants).

Commit `feat(sym): integration core — table, linearity, power rule`.

### Task 4: integration — u-substitution, parts, partial fractions

Heuristics tried in order after Task 3 core fails:

1. **u-substitution:** for products f(g(x))·h(x): if h = c·g′ (check: expand(h/g′) is constant — via poly quotient or eval at points then verify), integrate table form of f in u, substitute back. Covers 2x·e^(x²), cos(x)·sin(x)^n, g′/g → log(g).
2. **Integration by parts** (bounded depth 2): u chosen by LIATE (log/inverse-trig first, then poly), ∫u dv = uv − ∫v du; recurse on the remainder with depth budget; covers x·e^x, x·sin(x), log(x)·1, x²·e^x (depth 2), atan(x)·1.
3. **Partial fractions** for p(x)/q(x), both poly in x (from_expr succeeds): proper-ize via divmod; factor q by solve_poly exact roots — require complete_exact with rational roots (linear factors, with multiplicity via repeated deflation) or irreducible quadratic factors with rational coefficients (from conjugate approx pairs recovered exactly via deflation when possible — v1: rational-root linear factors + one leftover irreducible quadratic). Terms: A/(x−r)^k → A·(x−r)^(1−k)/(1−k) or A·log(x−r); (Bx+C)/(x²+bx+c) → (B/2)log(x²+bx+c) + adjusted-atan term using completed square.
   ∫1/(1+x²) = atan(x) falls out of the quadratic case.

Tests (diff-roundtrip oracle everywhere; the oracle avoids domain issues by sampling x in (safe interval per test)):
- u-sub: ∫2x·e^(x²) = e^(x²); ∫cos x · sin²x = sin³x/3; ∫(2x+1)/(x²+x+3) = log(x²+x+3).
- parts: ∫x e^x = (x−1)e^x; ∫x sin x = sin x − x cos x; ∫log x = x log x − x; ∫atan x = x atan x − log(1+x²)/2; ∫x² e^x (depth 2).
- partial fractions: ∫1/(x²−1) = (log(x−1)−log(x+1))/2; ∫1/(1+x²) = atan(x); ∫(x³+1)/(x−1) improper → poly part + log; ∫1/(x²+2x+5) → atan form with completed square; ∫1/(x−2)² = −1/(x−2).
- still-honest failures: ∫e^(x²), ∫sin(x)/x → nullopt.

Commit `feat(sym): integration heuristics — u-substitution, parts, partial fractions`.

### Task 5: equation solving

API (append to `solve.hpp`):

```cpp
/** Solutions of lhs == rhs for x. Returned exprs are verified exact
    solutions; empty vector = none found (not "no solutions exist").
    Handles: polynomial equations (via solve_poly, exact part), single
    unary-fn isolation f(u(x)) == c for invertible f (exp/log/sqrt with
    domain-valid c; sin/cos/tan principal solution), and linear-in-x
    equations with symbolic coefficients. */
std::vector<expr> solve(const expr& lhs, const expr& rhs, const expr& x);

/** Solve the n x n symbolic linear system A x = b by fraction-free Gaussian
    elimination over expr. Throws std::invalid_argument on shape mismatch,
    std::domain_error when a pivot (and all row swaps) is structurally zero. */
std::vector<expr> solve_linear_system(const std::vector<std::vector<expr>>& a,
                                      const std::vector<expr>& b);
```

Isolation rules (principal values, documented): exp(u)=c → u=log c (c must not be a non-positive constant); log(u)=c → u=e^c; sqrt(u)=c → u=c² (c≥0 when constant); sin(u)=c → u=asin c; cos(u)=c → u=acos c; tan(u)=c → u=atan c; then recurse solving u(x)=rhs′ (u linear or polynomial).

Tests:
- polynomial: solve(x²−4, 0) = {−2, 2} (verified by subs+eval).
- symbolic linear: solve(a·x+b, 0) = {−b/a} structurally.
- fn isolation: solve(exp(x)−5, 0) = {log 5}; solve(sin(2x), 1) = {asin(1)/2} eval ≈ π/4; solve(log(x+1), 0) = {0}; chained: solve(exp(2x+1), e²·e? ) — use exp(2x+1)=1 → x=−1/2.
- linear system 2x2 rational: {{2,1},{1,3}} b={5,10} → x={1,3}; symbolic 2x2 {{a,0},{0,b}} b={c,d} → {c/a, d/b}; singular {{1,1},{1,1}} throws std::domain_error.
- solve(sin(x)·x, 0) (not isolatable v1) → empty vector.

Commit `feat(sym): equation solving — polynomial, fn isolation, symbolic linear systems`.
