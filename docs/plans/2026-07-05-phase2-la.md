### Task 1: mat/vec container + element ops

**Files:**
- Create: `include/ax/la/mat.hpp`
- Test: `tests/la/mat_test.cpp` (add to tests/CMakeLists.txt)

Tests: construction (rows/cols/fill, initializer list of rows, identity), `at()` bounds throw `std::out_of_range`, `operator()(i,j)` unchecked access, add/sub/scalar-mul, transpose, mat-vec product (small known values), equality-within-tolerance helper `approx_equal(a, b, tol)`, shape mismatch throws `std::invalid_argument`.

Header contents:

```cpp
#pragma once
/** @file mat.hpp Dense row-major matrix and vector. */
#include <cstddef>
#include <initializer_list>
#include <stdexcept>
#include <vector>

namespace ax::la {

/** Dense vector of double. Thin wrapper over std::vector with math ops. */
class vec {
 public:
  vec() = default;
  explicit vec(std::size_t n, double fill = 0.0) : d_(n, fill) {}
  vec(std::initializer_list<double> xs) : d_(xs) {}

  std::size_t size() const noexcept { return d_.size(); }
  double& operator[](std::size_t i) noexcept { return d_[i]; }
  double operator[](std::size_t i) const noexcept { return d_[i]; }
  double* data() noexcept { return d_.data(); }
  const double* data() const noexcept { return d_.data(); }

  friend vec operator+(const vec& a, const vec& b);   // size mismatch throws
  friend vec operator-(const vec& a, const vec& b);
  friend vec operator*(double s, const vec& a);
  friend double dot(const vec& a, const vec& b);
  /// Euclidean norm.
  double norm() const;

 private:
  std::vector<double> d_;
};

/** Dense row-major matrix of double. */
class mat {
 public:
  mat() = default;
  mat(std::size_t rows, std::size_t cols, double fill = 0.0)
      : r_(rows), c_(cols), d_(rows * cols, fill) {}
  /// From rows; all rows must be equal length (throws std::invalid_argument).
  mat(std::initializer_list<std::initializer_list<double>> rows);
  /// n x n identity.
  static mat identity(std::size_t n);

  std::size_t rows() const noexcept { return r_; }
  std::size_t cols() const noexcept { return c_; }
  /// Unchecked element access.
  double& operator()(std::size_t i, std::size_t j) noexcept {
    return d_[i * c_ + j];
  }
  double operator()(std::size_t i, std::size_t j) const noexcept {
    return d_[i * c_ + j];
  }
  /// Checked element access; throws std::out_of_range.
  double& at(std::size_t i, std::size_t j);
  double at(std::size_t i, std::size_t j) const;
  double* data() noexcept { return d_.data(); }
  const double* data() const noexcept { return d_.data(); }

  mat transposed() const;

  friend mat operator+(const mat& a, const mat& b);
  friend mat operator-(const mat& a, const mat& b);
  friend mat operator*(double s, const mat& a);
  friend vec operator*(const mat& a, const vec& x);  // mat-vec

 private:
  std::size_t r_ = 0, c_ = 0;
  std::vector<double> d_;
};

/// True iff same shape and max |a-b| element diff <= tol.
bool approx_equal(const mat& a, const mat& b, double tol);
bool approx_equal(const vec& a, const vec& b, double tol);

}  // namespace ax::la
```

All implementations inline in header (small loops). Steps: failing tests → header → green → commit `feat(la): mat/vec containers and element ops`.

### Task 2: matmul — naive serial + blocked parallel

**Files:**
- Create: `include/ax/la/blas.hpp`, `src/la/blas.cpp` (add to CMake)
- Test: `tests/la/matmul_test.cpp`

API:

```cpp
#pragma once
/** @file blas.hpp Matrix multiplication kernels. */
#include <ax/la/mat.hpp>
#include <ax/par/pool.hpp>

namespace ax::la {

/** c = a * b, serial reference (i-k-j loop order). Shape mismatch throws. */
mat matmul(const mat& a, const mat& b);

/** c = a * b, cache-blocked, rows parallelized across pool. */
mat matmul(const mat& a, const mat& b, thread_pool& pool);

/// operator* uses the serial kernel.
inline mat operator*(const mat& a, const mat& b) { return matmul(a, b); }

}  // namespace ax::la
```

Implementation notes (src/la/blas.cpp):
- Serial: i-k-j order (streams b rows, autovectorizes), `const double aik = a(i,k)` hoisted.
- Blocked: tile `i` by 64 rows per task; inside task use i-k-j on full k/j (row-major b already streams); dispatch via `parallel_for(pool, 0, nblocks, 1, fn)`.
- Tests: known 2x3 * 3x2 product; random 65x37 * 37x53 serial vs parallel `approx_equal(tol=1e-12*k)`; A*I == A; shape mismatch throws; parallel matches serial on 200x200.

Steps: failing tests → implement → green → commit `feat(la): serial and pool-parallel blocked matmul`.

### Task 3: LU decomposition (partial pivot), solve, det, inverse

**Files:**
- Create: `include/ax/la/decomp.hpp`, `src/la/decomp.cpp` (add to CMake)
- Test: `tests/la/lu_test.cpp`

API (decomp.hpp starts here, later tasks append):

```cpp
#pragma once
/** @file decomp.hpp Dense decompositions: LU, Cholesky, QR. */
#include <ax/la/mat.hpp>

#include <cstdint>
#include <vector>

namespace ax::la {

/** PA = LU factorization with partial pivoting. */
struct lu_result {
  mat lu;                        ///< L (unit diag, strict lower) + U packed.
  std::vector<std::size_t> piv;  ///< row permutation: row i of PA = row piv[i] of A.
  int sign;                      ///< permutation parity (+1/-1).
  bool singular;                 ///< true if a zero pivot was hit.
};

/** Factor square a. Throws std::invalid_argument if not square. */
lu_result lu_decompose(const mat& a);
/** Solve a x = b via LU. Throws std::domain_error if singular. */
vec lu_solve(const lu_result& f, const vec& b);
/** Determinant via LU (0 for singular). */
double det(const mat& a);
/** Inverse via LU solve per unit vector. Throws std::domain_error if singular. */
mat inverse(const mat& a);

}  // namespace ax::la
```

Tests: 3x3 known det (e.g. {{2,1,1},{1,3,2},{1,0,0}} det=-3... verify by hand: expand along last row: 1*C31; compute in test with hardcoded expected from cofactor by hand — implementer: use {{1,2},{3,4}} det=-2 and {{6,1,1},{4,-2,5},{2,8,7}} det=-306, both standard); solve on random well-conditioned 50x50 (A = M + 50*I): `(a*x - b).norm() <= 1e-9`; `a * inverse(a) ~ I (tol 1e-9)`; singular matrix (duplicate rows) → det 0, lu_solve throws; non-square throws.

Steps: failing tests → implement (Doolittle in-place, pivot on max |column| below diag, forward/back substitution) → green → commit `feat(la): LU with partial pivoting, solve, det, inverse`.

### Task 5: Householder QR + least squares

**Files:**
- Modify: `include/ax/la/decomp.hpp`, `src/la/decomp.cpp`
- Test: `tests/la/qr_test.cpp`

API append:

```cpp
/** Householder QR: A (m x n, m >= n) = Q R with Q m x m orthogonal,
    R m x n upper trapezoidal. Throws std::invalid_argument if m < n. */
struct qr_result {
  mat q;  ///< orthogonal, m x m
  mat r;  ///< upper trapezoidal, m x n
};
qr_result qr_decompose(const mat& a);
/** Least squares min ||a x - b|| via QR (full column rank assumed).
    Throws std::domain_error on rank deficiency (zero diagonal in R). */
vec lstsq(const mat& a, const vec& b);
```

Tests: QᵀQ ~ I (tol 1e-10*m) and Q*R ~ A on random 30x20; R upper-trapezoidal (below-diag entries |.| < 1e-12); square system solve matches lu_solve; overdetermined fit: generate y = 2x + 1 + 0 noise over 50 points, lstsq recovers slope 2 intercept 1 to 1e-10; m < n throws.

Commit `feat(la): Householder QR and least squares`.
