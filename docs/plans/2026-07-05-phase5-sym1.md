# axiom Phase 5 Implementation Plan (ax::sym part 1 — CAS core)

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:executing-plans. TDD: RED before implementation on every task.

**Goal:** Symbolic engine core: immutable hash-consed expression DAG, canonicalizing simplifier, differentiation, univariate polynomial algebra over rationals, text/LaTeX printers.

**Architecture:** `expr` is an immutable shared handle (`std::shared_ptr<const node>`) into a global hash-consing pool — structurally identical subtrees share one node, so equality of handles = structural equality (fast). Node kinds: `num` (rational), `sym` (name), `add` (n-ary, flattened, sorted), `mul` (n-ary, flattened, sorted), `pow` (base, exp), `fn` (named unary: sin, cos, exp, log, ...). Construction functions (`make_add` etc.) canonicalize on build — there is no separate "unsimplified" state for the core rules; `simplify()` re-canonicalizes after substitutions and applies extra rewrite rules. Rationals via `ax::rational` (exact). Polynomials: `poly` = dense `std::vector<rational>` coefficient vector, lowest degree first.

**Ordering:** total order on exprs (kind rank, then payload) gives deterministic canonical form: `add` sorts terms, `mul` sorts factors, like terms/factors merge by coefficient/exponent arithmetic.

**Tech Stack:** C++23 STL, GoogleTest, ax::core rational.

**File map:**
- `include/ax/sym/expr.hpp` + `src/sym/expr.cpp` — node, pool, constructors, canonicalization, ordering, substitution, evaluation
- `include/ax/sym/calc.hpp` + `src/sym/calc.cpp` — differentiation
- `include/ax/sym/poly.hpp` + `src/sym/poly.cpp` — univariate poly over rationals: arithmetic, divmod, gcd, square-free, rational roots, expr↔poly conversion
- `include/ax/sym/print.hpp` + `src/sym/print.cpp` — to_string, to_latex
- Tests: `tests/sym/expr_test.cpp`, `calc_test.cpp`, `poly_test.cpp`, `print_test.cpp`

### Task 1: expr DAG + hash-consing + canonicalization

API (`namespace ax::sym`):

```cpp
class expr {  // immutable value handle; cheap copy
 public:
  // factories
  static expr num(std::int64_t n);
  static expr num(const rational& q);
  static expr symbol(std::string name);
  // observers
  bool is_num() const; bool is_sym() const; bool is_add() const;
  bool is_mul() const; bool is_pow() const; bool is_fn() const;
  const rational& value() const;        // num only (std::logic_error else)
  const std::string& name() const;      // sym/fn only
  std::span<const expr> args() const;   // add/mul: terms; pow: {base,exp}; fn: {arg}
  bool same(const expr& o) const;       // structural equality (pointer eq)
  std::size_t hash() const;
  // algebra (canonicalizing constructors)
  friend expr operator+(const expr&, const expr&);
  friend expr operator-(const expr&, const expr&);
  friend expr operator*(const expr&, const expr&);
  friend expr operator/(const expr&, const expr&);  // a * b^-1
  friend expr operator-(const expr&);
  expr pow(const expr& e) const;
  // functions
  static expr fn(std::string name, expr arg);  // sin/cos/tan/exp/log/sqrt...
  // ops
  expr subs(const expr& sym, const expr& replacement) const;
  double eval(const std::map<std::string, double>& env = {}) const;
  // std::logic_error on free symbol without binding
};
expr simplify(const expr& e);  // fixpoint of canonicalization + rules
```

Canonical rules at construction: flatten nested add/mul; fold numeric constants; collect like terms (`x+x → 2x`), like factors (`x*x → x^2`); drop identities (`+0`, `*1`); `*0 → 0`; `x^0 → 1`, `x^1 → x`, `num^num` folded when exact (integer exponent); sort operands by total order.

Tests: `x+x` same-node as `2*x`; `x*x*x` same as `x.pow(3)`; `(x+y)+(z+x)` = `2x+y+z`; constant folding `num(2)+num(3)` is `num(5)`; `2/3` stays exact rational; `x - x` = 0; `x/x` = 1; hash-consing: two independently built `x+y` are `same()`; subs: `(x^2+y).subs(x, num(3))` = `9+y`; eval `sin` etc. numeric vs std; eval unbound symbol throws; simplify(sin(x)^2 + cos(x)^2) stays put v1 (documents non-goal — no trig identities yet).
Commit `feat(sym): hash-consed expression DAG with canonicalization`.

### Task 2: differentiation

```cpp
expr diff(const expr& e, const expr& symbol);  // d e / d symbol
```

Rules: constants→0, dx/dx→1, linearity, product rule (n-ary), power rule (both `x^n` and general `u^v` via exp/log form when v non-constant), chain rule for fn (sin→cos, cos→-sin, tan→1+tan², exp→exp, log→1/x, sqrt→1/(2√x)).

Tests: d(x³+2x)/dx = 3x²+2 (structural); d(sin(x)cos(x)) = cos²−sin²... expressed as `cos(x)^2 - sin(x)^2` after simplify; d(exp(x²)) = 2x·exp(x²); d(log(x))=x⁻¹; d w.r.t. y of x = 0; numeric cross-check: eval(diff(f)) ≈ central difference of eval(f) at random points for 6 sample expressions (tol 1e-6) — the strongest oracle, no structural guessing.
Commit `feat(sym): symbolic differentiation`.

### Task 3: polynomial algebra

```cpp
class poly {  // univariate, exact rational coefficients, dense
 public:
  poly();                                  // zero
  explicit poly(std::vector<rational> coeffs);  // c0 + c1 x + ...
  static poly from_expr(const expr& e, const expr& sym);  // throws if not poly in sym
  expr to_expr(const expr& sym) const;
  int degree() const;                      // -1 for zero poly
  const rational& coeff(std::size_t k) const;
  friend poly operator+(const poly&, const poly&);
  friend poly operator-(const poly&, const poly&);
  friend poly operator*(const poly&, const poly&);
  std::pair<poly, poly> divmod(const poly& d) const;  // quotient, remainder
  friend poly gcd(poly a, poly b);         // monic gcd
  poly derivative() const;
  poly square_free() const;                // p / gcd(p, p')
  std::vector<rational> rational_roots() const;  // exact, with multiplicity collapsed
  rational eval(const rational& x) const;
};
```

Tests: ring identities on random small polys ((a+b)c = ac+bc, deg add/mul); divmod: (x³−1)/(x−1) = x²+x+1 rem 0; gcd(x²−1, x³−1) = x−1 (monic); square_free((x−1)²(x+2)) = (x−1)(x+2) up to constant; rational_roots(6x³−5x²−2x+1) = {1, 1/3, −1/2} (verify via eval == 0 too); from_expr((x+1)^2) coeffs {1,2,1}; from_expr(sin(x)) throws; to_expr∘from_expr roundtrip structural.
Commit `feat(sym): univariate polynomial algebra over rationals`.

### Task 4: printers

```cpp
std::string to_string(const expr& e);  // "2*x + y^2 - 3"
std::string to_latex(const expr& e);   // "2 x + y^{2} - 3", \frac for div, \sin etc.
```

Precedence-aware parenthesization; negative leading coefficients render as subtraction; rational exponents/coeffs as fractions in latex (`\frac{2}{3}`).

Tests: to_string(2x+y²−3) exact string; parens: (x+y)*z → "(x + y)*z"; x^(y+1) → "x^(y + 1)"; latex: \frac{2}{3} x, x^{2}, \sin\left(x\right); roundtrip sanity: strings non-empty, deterministic across two builds of same expr.
Commit `feat(sym): text and LaTeX printers`.
