# axiom Phase 6 Implementation Plan (ax::st part 2 — inference, regression, MCMC, time series)

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:executing-plans. TDD: RED before implementation on every task.

**Goal:** Statistical inference layer on top of Phase 3/4 foundations: classical hypothesis tests, OLS with full inference, GLMs (logistic, Poisson) via IRLS, Metropolis-Hastings MCMC, and time-series basics (ACF/PACF, AR/ARMA fitting, periodogram).

**Architecture:** Pure functions over `std::span<const double>` / `ax::la::mat`/`vec`, returning small result structs (same style as `ax::num`). Distribution CDFs from Phase 3 (`t_dist`, `chi2_dist`, `f_dist`, `normal_dist`) supply p-values. OLS rides `la::qr_decompose`/`lstsq`; IRLS solves weighted normal equations via `la::cholesky`. MCMC takes a user log-density `std::function` and `ax::st::rng`. Periodogram uses `ax::core` FFT. **SVD/eigen deferred**: QR + Cholesky cover everything Phase 6 needs (spec says "as needed" — it is not needed).

**Tech Stack:** C++23 STL, GoogleTest, existing ax::la / ax::st / ax::num / ax::core.

### Task 1: hypothesis tests

API (`namespace ax::st`):

```cpp
enum class alternative { two_sided, less, greater };

struct test_result {
  double statistic;  ///< test statistic
  double df;         ///< degrees of freedom (df1 for F; NaN where n/a e.g. KS)
  double df2;        ///< second df (F/ANOVA only; NaN otherwise)
  double p_value;
};

/** One-sample t: H0 mean == mu0. n >= 2 (std::invalid_argument). */
test_result t_test(std::span<const double> xs, double mu0,
                   alternative alt = alternative::two_sided);
/** Two-sample t. welch=true (default): Welch-Satterthwaite df;
    welch=false: pooled variance. Each sample n >= 2. */
test_result t_test(std::span<const double> xs, std::span<const double> ys,
                   alternative alt = alternative::two_sided, bool welch = true);
/** Chi-square goodness of fit. observed: counts; expected_p: probabilities
    summing to ~1 (renormalized). sizes match, >= 2 cells, all expected
    counts > 0 (std::invalid_argument). df = k-1. Two-sided by nature. */
test_result chi2_gof(std::span<const double> observed,
                     std::span<const double> expected_p);
/** Chi-square test of independence on an r x c contingency table of counts.
    r,c >= 2; all row/col sums > 0. df = (r-1)(c-1). */
test_result chi2_independence(const la::mat& table);
/** One-way ANOVA. >= 2 groups, each n >= 2. statistic = F,
    df = k-1, df2 = N-k. */
test_result anova_oneway(std::span<const std::vector<double>> groups);
/** One-sample Kolmogorov-Smirnov vs continuous CDF. statistic = D_n,
    p via asymptotic Kolmogorov series Q(sqrt(n)*D) (good for n >= ~10;
    documented). df = NaN. n >= 1. */
test_result ks_test(std::span<const double> xs,
                    const std::function<double(double)>& cdf);
```

p-values: t via `t_dist(df).cdf`, chi2 via `chi2_dist`, F via `f_dist`; one-sided/two-sided mapping from `alternative`. KS: `Q(x) = 2 * sum_{k>=1} (-1)^{k-1} exp(-2 k^2 x^2)` truncated when term < 1e-12.

Tests (oracles = R/scipy values computed by hand from published examples, quoted in test comments):
- one-sample t on {5.1,4.9,5.0,5.2,4.8} vs mu0=5.0: t=0, p=1 (symmetric); vs mu0=4.8: known t/p to 1e-6.
- Welch vs pooled on unequal-variance samples differ; pooled df = n1+n2-2 exact; Welch df fractional, matches hand formula.
- one-sided: p_less + p_greater = 1 (continuous statistic); two_sided = 2*min for symmetric t.
- chi2_gof fair-die counts {8,9,19,5,8,11}: chi2 = 13.2, df 5, p ≈ 0.0216 (hand-computable).
- chi2_independence 2x2 {{10,20},{30,40}}: chi2 vs hand formula, df=1.
- anova_oneway on 3 groups with equal means: F small, p > 0.5; textbook example with known F to 1e-6.
- ks_test of uniform sample (rng, n=1000) vs uniform cdf: p > 0.01; vs normal cdf: p < 1e-6.
- validation: single-element sample throws; ragged/empty groups throw.

Commit `feat(st): hypothesis tests (t, chi2, ANOVA, KS)`.

### Task 2: OLS regression

API (`namespace ax::st`, header `reg.hpp`):

```cpp
struct ols_result {
  la::vec beta;       ///< coefficients (intercept first if fitted)
  la::vec stderrs;    ///< standard errors
  la::vec t_stats;    ///< beta / stderr
  la::vec p_values;   ///< two-sided vs t(df_resid)
  la::vec residuals;  ///< y - X beta
  double sigma2;      ///< RSS / df_resid
  double r2, adj_r2;
  std::size_t df_resid;
};

/** Fit y = X beta (+ intercept column prepended when intercept=true).
    Requires rows > cols(+1), full column rank (std::domain_error via lstsq),
    size match (std::invalid_argument). */
ols_result ols(const la::mat& x, const la::vec& y, bool intercept = true);
```

Implementation: build design matrix (prepend ones column if intercept), `qr_decompose`, beta by back-substitution on R (reuse `lstsq`), covariance of beta = sigma2 * (R'R)^{-1} computed by inverting the upper-triangular n×n block of R (back-substitution per column — no general `inverse()` on X'X, better conditioning). R² relative to mean when intercept fitted, relative to zero otherwise (documented).

Tests:
- exact fit: y perfectly linear in x → residuals ~ 0, R² = 1, sigma2 ~ 0.
- hand oracle: small dataset (n=5, 1 predictor) with hand-computed beta, stderr, t, p (values in comments, tol 1e-8).
- multivariate: 2 predictors + intercept vs normal-equation solution computed in-test via `inverse(XtX)*Xt*y` (cross-check between two code paths, tol 1e-10).
- no-intercept flag: beta size = cols, R² definition switch verified.
- recovery: simulate y = 2 + 3x + noise (rng seed fixed, n=200); beta within 3 stderr of truth; p-value of slope < 1e-6.
- rank-deficient X (duplicated column) throws std::domain_error.

Commit `feat(st): OLS regression with inference via QR`.

### Task 3: GLM (logistic, Poisson) via IRLS

API (append to `reg.hpp`):

```cpp
enum class glm_family { logistic, poisson };

struct glm_result {
  la::vec beta;
  la::vec stderrs;   ///< from inverse Fisher information (X' W X)^{-1}
  double deviance;
  std::size_t iters;
  bool converged;
};

/** Fit GLM with canonical link (logit / log) by IRLS.
    logistic: y in {0,1}; poisson: y >= 0 integer-valued (validated).
    intercept prepended when intercept=true. Converged when max |delta beta|
    < tol or iter == max_iter (result.converged=false then — no throw,
    caller decides). Perfect separation surfaces as non-convergence. */
glm_result glm_fit(const la::mat& x, const la::vec& y, glm_family family,
                   bool intercept = true, std::size_t max_iter = 50,
                   double tol = 1e-10);
```

IRLS iteration: eta = X beta; mu = link⁻¹(eta); W = diag(mu(1-mu)) (logistic) or diag(mu) (poisson); z = eta + (y-mu)/W; solve (X' W X) delta-system via `cholesky` on X'WX (SPD for full-rank X, W > 0). Clamp mu to [1e-10, 1-1e-10] (logistic) / [1e-10, inf) (poisson) for stability. Deviance: logistic 2Σ[y log(y/mu) + (1-y) log((1-y)/(1-mu))] (0-terms dropped); poisson 2Σ[y log(y/mu) − (y−mu)].

Tests:
- logistic hand oracle: tiny dataset (n=10) with beta/stderr from R `glm()` quoted in comments (tol 1e-6).
- logistic recovery: simulate n=500, beta_true=(−1, 2), fitted within 3 stderr; converged=true, iters < 25.
- poisson hand oracle: small counts dataset vs R `glm(family=poisson)` (tol 1e-6).
- poisson recovery: simulate log-link data, recover beta.
- null model (intercept only) logistic: beta0 = logit(ȳ) exact to 1e-8.
- perfect separation: converged=false, no throw, no NaN in beta.
- y outside {0,1} for logistic throws std::invalid_argument.

Commit `feat(st): logistic and Poisson GLM via IRLS`.

### Task 4: MCMC (Metropolis-Hastings)

API (`namespace ax::st`, header `mcmc.hpp`):

```cpp
struct mh_options {
  std::size_t n_samples = 1000;  ///< kept samples (post burn-in, post thin)
  std::size_t burn_in = 1000;
  std::size_t thin = 1;          ///< keep every thin-th draw
  double step = 1.0;             ///< isotropic gaussian proposal sd
};

struct mh_result {
  la::mat samples;         ///< n_samples x dim
  double acceptance_rate;  ///< over all post-burn-in proposals
};

/** Random-walk Metropolis with N(0, step^2 I) proposals.
    log_density: unnormalized log target; may return -inf for out-of-support
    (always rejected). x0 must have finite log_density
    (std::invalid_argument). thin >= 1, n_samples >= 1. */
mh_result metropolis(const std::function<double(const la::vec&)>& log_density,
                     const la::vec& x0, const mh_options& opt, rng& g);

/** Effective sample size of one chain component via initial positive
    sequence estimator on the autocorrelation (Geyer). */
double ess(std::span<const double> chain);
```

Tests (fixed seeds, statistical tolerances chosen loose enough to be deterministic-stable):
- 1-d standard normal target, 20k samples: sample mean |m| < 0.05, var in [0.9, 1.1].
- 2-d correlated gaussian (rho=0.8) target: sample correlation in [0.7, 0.9].
- bounded support: log_density −inf outside (0,1), all samples inside.
- acceptance_rate in (0,1); tiny step → rate > 0.9; huge step → rate < 0.2.
- determinism: same seed twice → identical sample matrices.
- ess: iid normal draws → ess ≈ n (within factor 2); strongly autocorrelated chain (AR(1) φ=0.95) → ess < n/5.
- x0 with -inf density throws.

Commit `feat(st): Metropolis-Hastings MCMC with ESS diagnostic`.

### Task 5: time series

API (`namespace ax::st`, header `ts.hpp`):

```cpp
/** Sample autocorrelation, lags 0..max_lag (acf[0] = 1).
    Denominator n (standard biased estimator). n > max_lag >= 0. */
std::vector<double> acf(std::span<const double> xs, std::size_t max_lag);
/** Partial autocorrelation, lags 1..max_lag, via Durbin-Levinson. */
std::vector<double> pacf(std::span<const double> xs, std::size_t max_lag);

struct ar_result {
  std::vector<double> phi;  ///< AR coefficients 1..p
  double intercept;         ///< mean-form constant
  double sigma2;            ///< innovation variance
};
/** Fit AR(p) by Yule-Walker on demeaned series. n > p (invalid_argument). */
ar_result ar_fit(std::span<const double> xs, std::size_t p);

struct arma_result {
  std::vector<double> phi, theta;
  double intercept, sigma2;
  bool converged;
};
/** Fit ARMA(p,q) by conditional sum of squares (CSS) minimized with
    nelder_mead; AR part initialized from Yule-Walker, MA from zeros.
    Innovations recursion with zero pre-sample values. */
arma_result arma_fit(std::span<const double> xs, std::size_t p, std::size_t q);

/** Raw periodogram at Fourier frequencies f_k = k/n, k = 1..n/2, via
    ax::core fft (series zero-padded to next power of two; frequencies
    reported for padded length; documented). Returns {frequency, power}. */
struct periodogram_result { std::vector<double> freq, power; };
periodogram_result periodogram(std::span<const double> xs);
```

Tests:
- acf of iid noise (fixed seed, n=2000): lag-0 = 1, |lag>=1| < 0.05.
- acf of AR(1) φ=0.8 simulation: acf[k] ≈ 0.8^k for k <= 5 (tol 0.08).
- pacf of AR(2): pacf[1], pacf[2] nonzero, |pacf[k>=3]| < 0.06 (cutoff property).
- pacf vs acf identity for AR(1): pacf[1] == acf[1] exactly (Durbin-Levinson step 1).
- ar_fit on simulated AR(2) φ=(0.5, −0.3), n=5000: phi within 0.05, sigma2 within 10%.
- arma_fit on simulated ARMA(1,1) φ=0.6 θ=0.4, n=4000: params within 0.1, converged=true; pure AR case agrees with ar_fit within 0.02.
- periodogram of pure sinusoid sin(2π·0.1·t), n=512: argmax power at freq nearest 0.1; white-noise periodogram roughly flat (max/mean ratio sanity bound).
- validation: max_lag >= n throws; ar_fit p >= n throws.

Commit `feat(st): time series — ACF/PACF, AR/ARMA fitting, periodogram`.
