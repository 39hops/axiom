## The audit caught a real axiom soundness bug (fixed)

Run 1 had 25 axiom-wrong rows, all one root cause: pow-of-pow
canonicalization collapsed `(x^a)^b → x^(a·b)` unconditionally, so
`(x²)^(1/2)` (= |x|) became `x`, and d/dx `sqrt(x**2)`-shaped terms came
out wrong for x < 0 (13 diff rows, 12 equiv_mod_const crossings). Fixed in
`fix(sym): restrict (x^a)^b exponent collapse to sound cases` (collapse
only for integer outer exponent or −1 ≤ inner ≤ 1); the re-run has zero
axiom-side disagreements. This is exactly what the audit was for.

## Reference-side findings: 64 crossings, all independently confirmed

In all 64 remaining crossings axiom says NOT_EQUIVALENT, the reference
says EQUIVALENT — and *plain Python `math` evaluation* (neither oracle)
confirms the two sides differ numerically in every single case, e.g.
`e1426`: `4*x*(...)` vs `x*(...)` — a factor-4 scalar mismatch the
reference labels EQUIVALENT. The recurring shape is `lhs = k·rhs` or a
single perturbed coefficient, which smells like the ref generator labeling
pairs by provenance (both derived from the same source row) rather than by
sympy equality, or an equality-mod-constant-factor check leaking in.
Findings, with per-row witness points, in `parity_ref_findings.jsonl` —
reported upstream, not papered over.
